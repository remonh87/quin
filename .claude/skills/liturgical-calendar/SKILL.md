# Liturgical Calendar Lookup

Answers questions about the Roman Catholic liturgical calendar — saints, feast
ranks, seasons, and liturgical colors — for a date, a range, a season, or a
named celebration. The data already exists as markdown tables in
`knowledge/liturgical-calendar/YYYY-MM.md` (one file per month, generated by
`scripts/generate-liturgical-calendar.js` from romcal's General Roman
Calendar). This skill's job is to read that data efficiently and answer the
question — not to regenerate everything from scratch each time.

Each month file has this shape, so the columns below are what you're matching
against:

```
| Date | Celebration | Rank | Season | Color |
|---|---|---|---|---|
| 2026-07-03 (Fri) | Saint Thomas, Apostle | Feast | Ordinary Time | Red |
```

Ranks in ascending liturgical importance: `Weekday` / `Sunday` < `Optional
Memorial` < `Memorial` < `Feast` < `Solemnity`. Optional memorials that don't
override the day are folded into the Celebration cell as `*(opt: ...)*` rather
than given their own row — keep that in mind when matching by name.

## Step 1: Work out which file(s) you need

Figure out the year(s) and month(s) the question touches:

- A single date → one `YYYY-MM.md` file.
- "This week" / "this month" → resolve relative to today's date (given in
  your session context), which may span two files if it crosses a month
  boundary.
- A named season (Advent, Christmas Time, Lent, The Sacred Paschal Triduum,
  Easter Time, Ordinary Time) → seasons don't fall on fixed calendar dates
  (they shift with Easter), so you generally need to scan **all 12 months of
  the relevant year** rather than guess a date range.
- A saint/celebration name → same as above, scan the relevant year's files. If
  Remon doesn't say which year, default to the current year; if the saint
  could plausibly be asked about across multiple years (e.g. "when is the
  feast of St. Francis" with no year context, and the feast date doesn't move
  year to year anyway), one year is enough to answer — you don't need to
  check every generated year.

## Step 2: Handle a missing file

If the file you need doesn't exist yet (e.g. Remon asks about a year that
hasn't been generated), don't just say "no data" — generate it, the same way
the project's readme documents:

```bash
npm run generate:liturgical-calendar -- <year>
```

Run this from the repo root. It's safe to re-run even for years that already
exist (it overwrites deterministically, no duplicates), so if you're ever
unsure whether a file is stale, regenerating is cheap and harmless.

## Step 3: Extract the rows you need

Prefer `grep` over reading a whole file into context — these tables are
designed to be scanned cheaply, and `working-style.md` asks for
token-conscious tool use:

- **Single date**: `grep "^| YYYY-MM-DD" knowledge/liturgical-calendar/YYYY-MM.md`
- **Range within one file**: `grep -E "^\| YYYY-MM-(0[5-9]|1[0-2])" ...` or
  just read the file if the range covers most of the month anyway.
- **Season**: `grep -i "| Advent " knowledge/liturgical-calendar/YYYY-*.md`
  (match against the Season column; note some season names are substrings of
  others, e.g. "Lent" vs "Lenten", so check the actual season names romcal
  uses — Christmas Time, Ordinary Time, Easter Time, Advent, Lent, The Sacred
  Paschal Triduum).
- **Name lookup**: `grep -i "francis" knowledge/liturgical-calendar/YYYY-*.md`
  across the relevant year's files.

## Step 4: Format the answer

- **Single date** → a short prose answer: the celebration name, its rank,
  season, and color. E.g. "July 3, 2026 is the feast of Saint Thomas, Apostle
  (Feast, Ordinary Time, red)." No need for a table for one row.
- **Range or season** → reproduce the relevant rows as a markdown table with
  the same columns (Date | Celebration | Rank | Season | Color), so it's easy
  to scan. If the range is long (e.g. all of Lent), lead with a one-line
  summary of the named Solemnities/Feasts/Memorials in that stretch before the
  full table — that's usually what "important events" means, while the table
  still gives Remon the complete picture including ordinary weekdays.
- **Name lookup** → state the date(s) it falls on this year, plus rank/season/
  color, the same as a single-date answer (a saint can have more than one
  entry across years if you checked multiple).

Don't pull in unrelated months "just in case" — read or grep only what the
question actually needs.
