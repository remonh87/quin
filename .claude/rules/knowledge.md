# Knowledge Base

There is a `knowledge/` directory in this project containing my Obsidian vault
(markdown notes, some with `[[wikilinks]]` and YAML frontmatter).

- Treat this as your primary source of truth about my ongoing projects, ideas,
  and reference notes — check it before asking me something I may have already
  written down.
- When reading notes, follow `[[wikilinks]]` to related notes if they're relevant
  to the task at hand, but don't recursively pull in the whole vault for a simple
  question.
- When asked to add or update knowledge, write in the same style as existing notes
  (frontmatter + concise markdown) rather than dumping unstructured text.
- Don't treat everything in the vault as current fact — some notes may be drafts,
  outdated, or exploratory; flag if something looks stale or contradicts what I
  just told you.
